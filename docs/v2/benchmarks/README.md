# V2 Tiered Benchmarks

Use this directory to store locked cutover threshold files per hardware tier.

Tracked tiers:

- `desktop_mid`
- `laptop_integrated`

## Workflow

1. Capture baseline and lock thresholds on each target tier host:

```bash
scripts/bench/tier_gate.sh lock <tier-name>
```

2. Validate future runs against the locked thresholds:

```bash
scripts/bench/tier_gate.sh validate <tier-name>
```

3. Enable hardware-tier gates in CI once both files are locked:

- Set repository variable `COVERGEN_ENABLE_HARDWARE_TIER_GATES=true`.
- Ensure self-hosted runners with labels exist:
  - `covergen-desktop-mid`
  - `covergen-laptop-integrated`
- CI workflow: `.github/workflows/perf-gates.yml`

## Artifact Notes

- Runtime report: `target/bench/<tier-name>/benchmark_report.md`
- Machine-readable metrics: `target/bench/<tier-name>/benchmark_metrics.ini`
- Locked thresholds: `docs/v2/benchmarks/<tier-name>.thresholds.ini`
- CI software-gate thresholds: `.github/bench/ci_software.thresholds.ini`

Threshold files are generated directly from measured baselines and include p50/p95 latency, frame time, and throughput bounds.
The committed hardware-tier files are placeholders until regenerated by `scripts/bench/tier_gate.sh lock <tier-name>` on the corresponding GPU host.
